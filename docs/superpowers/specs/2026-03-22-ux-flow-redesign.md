# UX Flow Redesign — TenderSarthi

**Date:** 2026-03-22
**Status:** Approved
**Scope:** Mobile navigation, adaptive dashboard, Find page cleanup

---

## Problem

The app has 9 screens but mobile users can only reach 5 of them. The bottom nav's "More" tab routes directly to `/settings` instead of a menu — a significant mislabelling that exposes users to the account deletion danger zone. New users land on a dashboard with no clear next action. The Find page stacks 5 unrelated UI sections with no visual hierarchy, and the Algolia saved-tender search sits on the wrong page entirely.

---

## Goals

1. Make all 9 screens reachable from mobile
2. Give new users (0 tenders) a clear guided first step
3. Give returning users (1+ tenders) an at-a-glance status view
4. Reduce Find page cognitive load — one clear job per page
5. Keep the desktop sidebar and all existing screen internals untouched

---

## What We Are Changing

### 1. Bottom Nav — "More" → "Menu"

**Current:** `Dashboard | Find | Tenders | Bid | More (→ /settings)`
**New:** `Home | Find | Tenders | Bid | Menu (↑ bottom sheet)`

The fifth tab label changes from "More" to "Menu". Tapping it opens a `MenuSheet` component (a bottom sheet/drawer) instead of navigating to `/settings`.

The bottom nav tabs for tabs 1–4 are unchanged in route and behaviour. Only the fifth tab changes.

### 2. MenuSheet Component (new)

A bottom sheet that slides up when the Menu tab is tapped. Closes on tap-outside or drag-down.

**Header strip:** user avatar (initials), display name, plan badge (Free / Pro Trial N days / Pro).

**2×3 grid of nav items:**
- Alerts (lock badge for free users, Pro-gated)
- Learn (free for all)
- Documents (lock badge for free users, Pro-gated)
- Orders (lock badge for free users, Pro-gated)
- Settings
- Log Out (destructive red — calls `signOut()`, redirects to `/{locale}/auth`)

Tapping any item closes the sheet and navigates to the target route. Pro-gated items navigate normally — the existing lock screens on those pages handle the gate.

**File:** `src/components/layout/menu-sheet.tsx`

### 3. Adaptive Dashboard

The dashboard detects user state using a single boolean: `tenders.length === 0`.

**New User State (`tenders.length === 0`):**

Replaces the feature cards section with a Getting Started checklist. Three steps, evaluated from existing data:

| Step | Label | Complete when |
|------|-------|---------------|
| 1 | Account बना लिया ✓ | Always true (they're logged in) |
| 2 | पहला Tender खोजें | `tenders.length > 0` |
| 3 | AI से Bid बनाएं | `usage.queries > 0` |

Step 2 is the active CTA — tapping it navigates to `/find`. Step 3 is locked (visually dimmed) until step 2 completes. Step 1 is always shown as done.

The trial banner, greeting, and AI usage counter remain visible above the checklist.

A contextual tip card below the checklist suggests the user's first category based on their onboarding selection.

**Active User State (`tenders.length > 0`):**

Replaces the old feature cards grid with:
- **Deadline card** — navy gradient, shows the next upcoming tender (name + closing date + days remaining). If no deadline is set, shows the most recently added tender.
- **Stats row** — three pills: Active count, Won count, Bids sent (total bid docs generated by this user).
- **Quick action grid** — 2×2 grid: Find Tenders → `/find`, Generate Bid → `/bid` (generator tab), Set Alerts → `/alerts`, Learn → `/learn`.

Once the dashboard switches to the active state (first tender saved), it never switches back to Getting Started.

**File:** `src/app/[locale]/(app)/dashboard/page.tsx` (modified)
**New component:** `src/components/dashboard/getting-started.tsx`
**New component:** `src/components/dashboard/active-dashboard.tsx`

### 4. Find Page Cleanup

**Removed from Find:** `AlgoliaSearch` component
**Added to Tenders page:** `AlgoliaSearch` component (at the top, above the tender list)

This clarifies the mental model: Find = discover new tenders, Tenders = manage your saved ones.

**Find page new layout (top to bottom):**

1. **Compact filter row** — state picker + category pills collapsed to a single horizontal row with the GeM deeplink button on the right. No separate "Filters" section heading.
2. **Live Tenders feed** (`GemLiveFeed`) — primary content, full width. Section label: "● Live Government Tenders" with an orange dot to signal live data.
3. **AI Summarizer** (`AISummarizer`) — collapsed into an accordion. Header: "🤖 AI Tender Summarizer — tap to expand". Expands inline on tap.

**Files modified:**
- `src/app/[locale]/(app)/find/page.tsx`
- `src/app/[locale]/(app)/tenders/page.tsx`
- `src/components/finder/state-category-filters.tsx` (compact row layout)

## Component Architecture

```
BottomNav
  └── tab 5: MenuSheet (new)
        ├── UserStrip (name, plan badge)
        └── NavGrid (6 items: Alerts, Learn, Documents, Orders, Settings, LogOut)

Dashboard (modified)
  ├── [tenders.length === 0] → GettingStarted (new)
  │     ├── StepItem × 3
  │     └── TipCard
  └── [tenders.length > 0]  → ActiveDashboard (new)
        ├── DeadlineCard
        ├── StatsRow
        └── QuickActionGrid

FindPage (modified)
  ├── CompactFilterRow (state + categories + GeM button, single row)
  ├── GemLiveFeed (existing, now primary)
  └── AISummarizerAccordion (existing, now collapsed)

TendersPage (modified)
  ├── AlgoliaSearch (moved from FindPage)
  └── [existing tender list + FAB]
```
